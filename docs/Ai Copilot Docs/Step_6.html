STEP 6 — Architect Agent

Max-Logic • Architecture Before Code • Deterministic Design

1. Position in the Execution Pipeline

STEP 1 → Intent Truth
STEP 2 → Context Snapshot
STEP 3 → Structural Validation
STEP 4 → Gaps & Inference
STEP 5 → Execution Plan
STEP 6 → Architecture & Code Mapping

STEP 6 converts the execution plan into a concrete, enforceable software architecture. No file is created and no code is written yet.

2. Core Truth

Code without architecture is technical debt in disguise.

The Architect Agent ensures that every future line of code has a justified place within a coherent system design.

3. Inputs

Inputs:
- ExecutionPlan (STEP 5)
- ContextSnapshot (STEP 2)
- CrossReferenceReport (STEP 3)
- InferenceReport (STEP 4)

4. What Architecture Means (Strict Definition)

Architecture is the explicit mapping between responsibilities and system boundaries.

  • Which component owns which responsibility
  • How components communicate
  • Where data flows and is transformed
  • What is allowed and forbidden across boundaries

5. Internal Architecture

Architect Agent
 ├─ Responsibility Mapper
 ├─ Module Boundary Designer
 ├─ Interface & Contract Definer
 ├─ Dependency Rule Enforcer
 ├─ Change Impact Analyzer
 └─ Architecture Validator

6. Module & Layer Design

The agent defines or adjusts layers without violating existing constraints. 

Typical Layers:
- API / Interface Layer
- Application / Service Layer
- Domain / Core Logic
- Infrastructure / Adapters

No layer mixing is allowed.

7. Interface & Contract Definition

Each interaction is formalized before coding begins. 

Contract Example:
- Input schema
- Output schema
- Error cases
- Side effects

8. Dependency Rules

Rules:
- High-level modules never depend on low-level details
- Infrastructure depends on core, never the inverse
- Cyclic dependencies are forbidden

9. Change Impact Analysis

Before any code exists, the agent predicts the blast radius of changes.

  • Affected modules
  • Backward compatibility risks
  • Testing implications

10. Output Structure

ArchitectureSpec {
  components: ComponentSpec[]
  interfaces: InterfaceSpec[]
  dependency_rules: string[]
  file_map: string[]
  change_risk: low | medium | high
}

11. Validation Rules

  • No component without responsibility
  • No implicit coupling
  • No architecture violating STEP 3 constraints

12. Output Guarantee

After STEP 6, every future line of code has a predefined place, purpose, and boundary.
