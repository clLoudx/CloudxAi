This is **the mandatory first step**. The installer automatically: 


Save the script as install.sh in your project root.

chmod +x install.sh

To run unattended: sudo ./install.sh --non-interactive

To debug interactively: sudo ./install.sh (it will prompt)

To resume a partial install: sudo ./install.sh --resume

To force longer retries for very flaky networks: sudo ./install.sh --ultra-mode --non-interactive

Logs: /var/log/ai-agent-install-v4.log

State file (resume): /var/lib/ai-agent/install_state.json

Backup copies of anything replaced: /opt/aiagent-backups/
  • Fixes broken requirements.txt
  • Creates a clean Python virtualenv
  • Installs missing pip modules
  • Installs Docker if missing
  • Deploys Redis via Docker
  • Prepares the dashboard environment
  • Validates everything with Max-Logic flow
chmod +x installer.sh
sudo ./installer.sh install

After installation, edit the global env: 

sudo nano /etc/ai-agent/env

Then start the full Agent stack: 

sudo ./installer.sh start

Step 2 â€” Optional Manual Local Development Setup

You only use this for debugging or modifying internals. The installer is still recommended even in dev.

git clone <repo> ai-agent
cd ai-agent

python3 -m venv venv
source venv/bin/activate

# auto-fix corrupted requirements
/bin/sed 's/\\\\n/\n/g' requirements.txt > clean_requirements.txt
mv clean_requirements.txt requirements.txt

pip install --upgrade pip setuptools wheel
pip install -r requirements.txt

docker run -d --name redis -p 6379:6379 redis:7

cd dashboard
python app.py

Dashboard will run on http://127.0.0.1:5000

Step 3 â€” Self-Repair Mode

Use this if something breaks, or after git pull:

sudo ./installer.sh repair

The repair system automatically fixes:

  • Corrupted requirements
  • Broken virtualenvs
  • Missing Python modules
  • Redis & Docker issues
  • File permissions
  • Service restarts
