B — STEP 1 Expanded: Input / UX / IDE (Core of Intelligence)

B1. Why STEP 1 is the Most Important

If input is wrong, reasoning is hallucination. If context is partial, execution is dangerous.

B2. Input Sources (Hard Enumerated)

  • User message (NL or command)
  • Open files (full content + hash)
  • Cursor location
  • Git diff + history
  • Terminal output (stdout + stderr)
  • Environment metadata

B3. Normalization Pipeline

Raw Input
 → Validation
 → Normalization
 → Canonical Context Object
 → Immutable Fact Store

B4. Canonical Input Object

{
  intent: "build api",
  scope: ["backend"],
  files: [...],
  diffs: [...],
  terminal: { cmd, output, code },
  risk_level: "medium",
  permissions: ["write", "execute"]
}

AI is forbidden to invent anything outside this object.

B5. UX & IDE Integration

  • Side panel = reasoning trace
  • Editor inline diffs
  • Terminal execution preview
  • Human approval checkpoints
