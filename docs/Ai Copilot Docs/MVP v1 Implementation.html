Rule: If it does not run → it does not exist

INPUT — What We Are Implementing

MVP v1 of an AI Copilot OS with:

  • One user
  • One project
  • One AI provider
  • Execution + Debug + Fix loop

PARSE — What MVP v1 Is (And Is Not)

MVP v1 is NOT:

  • Full SaaS
  • Multi-user system
  • Billing platform

MVP v1 IS:

  • A working AI execution engine
  • A deterministic flow from input → output

CROSS-REFERENCE — Proven Minimal Systems

  • Unix tools → small, composable programs
  • Docker → reproducible execution
  • FastAPI → minimal async backend

INFER — Required Components

From STEP 1 & 2, MVP v1 requires exactly these components:

  • API Server
  • Intent Parser
  • AI Router
  • Execution Engine (CMD / FS)
  • Result Analyzer

TEST — Runtime Flow (Non-Negotiable)

User Input
  → Intent Parsing
  → Context Build
  → AI Routing
  → Execution Plan
  → Command Execution
  → Error?
      → Yes → Send error to AI → Fix → Re-run
      → No  → Final Output

If this loop is broken, the MVP has failed.

EXECUTION — Terminal Reality

Bootstrap

mkdir ai-copilot-os
cd ai-copilot-os
mkdir backend frontend

Run

docker compose up

Test

curl http://localhost:8000/chat

VALIDATE — Success Criteria

  • AI writes code
  • Code runs
  • Error appears
  • AI fixes it
  • Code runs successfully
